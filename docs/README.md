# Docs

This is the documentation hub for agentic-playbook. The repo separates **rationale** (why the architecture looks the way it does) from **evidence** (what we read before deciding) from **artifacts** (the generated knowledge graph and the security ledger).

## Where to find what

| Subdirectory | What lives here | When to consult |
|---|---|---|
| [`adr/`](adr/README.md) | 49 Architecture Decision Records. Why each design choice was made, what the alternatives were, what the consequences are. | When you want to understand "why does the installer use X instead of Y." |
| [`research/`](research/README.md) | Dated research briefs that informed the ADRs. Inspirations map, failure-modes catalog, upcoming-adapters tracking. | When you want the evidence base behind a design decision, or when you're proposing a new adapter / pattern. |
| [`atlas/`](atlas/README.md) | Auto-generated knowledge graph (HTML) that links ADRs, skills, trajectories, and their cross-references. Per ADR-0049. | When you want to browse the corpus visually rather than file-by-file. Open `atlas/index.html` in a browser. |
| [`security/`](security/README.md) | AI Bill of Materials (`ai-bom.json`) and the supply-chain gate's output. Per ADR-0047. | When you want to know what external skills are imported, what their pinned source is, and when they were last vetted. |
| [`tools/`](tools/) | Per-agent integration notes (Tier 1 / 2 / 3 specifics, hooks contracts, MCP transports). | When you're adding a new adapter or troubleshooting an existing one. |
| `human-html/` | HTML artifacts (plans, reviews, decision aids, audits) authored by humans or coding agents. Gitignored except for `index.html` + `GLOSSARY.md` + a curated audit set. | When you want to read a specific session's plan / audit / explainer; the `index.html` auto-indexes the rest. |

## Doc generators

Some content under `docs/` is auto-generated by scripts; do not hand-edit:

- **`docs/atlas/`** is regenerated by `make atlas` (`scripts/build_atlas.py` per ADR-0049). Edits will be overwritten.
- **`docs/security/ai-bom.json`** is regenerated by `make check` and `make audit` (`scripts/security/ai_bom.py` per ADR-0047). Edits will be overwritten.

Everything else (`adr/`, `research/`, `tools/`, `human-html/`) is hand-authored.

## How docs relate to the code

- **ADRs land in code as gates.** When ADR-0036 (three-layer content contract) shipped, `scripts/install_verify.py` and `make doctor-verify` landed alongside it. ADR-0047 (supply-chain gate) shipped with `scripts/security/`. The ADR explains why; the code enforces.
- **Research lands in ADRs as evidence.** `research/inspirations.md` cites the source repos (mattpocock/skills, Block/Goose, Stripe Minions, Anthropic skills bundle). ADRs that borrow from those sources cite the research brief in their `## Context` section.
- **Atlas reads everything else.** The knowledge graph at `atlas/index.html` cross-references every ADR, skill, trajectory, BOM component, and telemetry signal. It's the "everything in one browse" view of the corpus.

## Reading order for a new contributor

1. Root [`README.md`](../README.md) for the elevator pitch.
2. [`CONTEXT.md`](../CONTEXT.md) for the canonical vocabulary (skill / rule / hook / adapter / profile / trajectory).
3. [`docs/adr/0036-three-layer-content-contract.md`](adr/0036-three-layer-content-contract.md) for the core contract every other piece of the system rests on.
4. [`docs/adr/0024-adapter-protocol-and-install-manifest.md`](adr/0024-adapter-protocol-and-install-manifest.md) for the per-tool extension model.
5. [`docs/adr/0044-trajectories-as-eighth-content-type.md`](adr/0044-trajectories-as-eighth-content-type.md) for the cross-adapter behavior testing story.
6. `make atlas` then open `atlas/index.html` for the visual map.
