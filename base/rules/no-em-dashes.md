# No Em Dashes

Never use em dashes (—) in any written prose: code comments, docstrings, PR descriptions, commit messages, documentation, Slack messages, Jira tickets, or HTML artifacts.

## Use instead

- Comma: "the agent works well, especially for backend tasks"
- Parentheses: "the agent works well (especially for backend tasks)"
- Separate sentences: "the agent works well. Backend tasks are where it shines."

## Why

Em dashes are a common LLM tell. The point of this rule is not to obscure that AI is involved, but to write more naturally and read like the team writes. team uses commas, parentheses, and separate sentences.
